# 테이블오더 서비스 - User Personas

## Persona 1: 디지털 네이티브 고객 (김민지, 28세)

### Profile
- **이름**: 김민지
- **나이**: 28세
- **직업**: IT 회사 직원
- **디지털 친숙도**: 높음

### 특성
- 스마트폰/태블릿 사용에 매우 익숙
- 빠른 주문과 효율성을 중시
- 새로운 디지털 서비스에 거부감 없음
- 메뉴 사진과 설명을 꼼꼼히 확인

### 목표
- 대기 시간 없이 빠르게 주문하고 싶다
- 메뉴를 편하게 탐색하고 비교하고 싶다
- 알러지 정보를 쉽게 확인하고 싶다

### 불편사항
- 직원을 불러서 주문하는 것이 번거로움
- 메뉴판이 오래되어 가격이 다른 경우
- 주문 상태를 알 수 없는 것

### 관련 스토리
- 메뉴 탐색 및 상세 조회
- 장바구니 관리
- 주문 생성 및 팁 추가
- 주문 내역 조회

## Persona 2: 디지털 비친숙 고객 (박영수, 62세)

### Profile
- **이름**: 박영수
- **나이**: 62세
- **직업**: 자영업자 (은퇴 예정)
- **디지털 친숙도**: 낮음

### 특성
- 태블릿 사용에 익숙하지 않음
- 큰 글씨와 명확한 버튼을 선호
- 단순하고 직관적인 인터페이스 필요
- 실수에 대한 불안감이 있음

### 목표
- 복잡하지 않게 음식을 주문하고 싶다
- 잘못 주문했을 때 쉽게 수정하고 싶다
- 주문이 제대로 들어갔는지 확인하고 싶다

### 불편사항
- 작은 버튼이나 복잡한 UI
- 실수로 잘못 터치하는 것
- 주문 과정이 너무 많은 단계를 거치는 것

### 관련 스토리
- 메뉴 탐색 (큰 카드, 명확한 가격)
- 장바구니 관리 (쉬운 수량 조절)
- 주문 생성 (확인 단계)

## Persona 3: 매장 사장님 (이준호, 45세)

### Profile
- **이름**: 이준호
- **나이**: 45세
- **직업**: 음식점 사장
- **디지털 친숙도**: 중간

### 특성
- 매장 운영 전반을 관리
- 효율적인 운영과 비용 절감에 관심
- 메뉴 구성과 가격 결정 권한 보유
- 바쁜 시간대에 빠른 판단이 필요

### 목표
- 주문을 실시간으로 파악하고 싶다
- 메뉴를 쉽게 추가/수정하고 싶다
- 테이블 회전율을 높이고 싶다
- 매장 운영 상황을 한눈에 보고 싶다

### 불편사항
- 주문이 밀려서 놓치는 경우
- 테이블 정리 후 이전 주문이 남아있는 경우
- 메뉴 변경이 즉시 반영되지 않는 경우

### 관련 스토리
- 매장 인증 및 로그인
- 실시간 주문 모니터링
- 테이블 세션 관리
- 메뉴 관리 (CRUD)
- 과거 주문 내역 조회

---

## Persona 4: 매장 직원 (최서연, 23세)

### Profile
- **이름**: 최서연
- **나이**: 23세
- **직업**: 음식점 아르바이트
- **디지털 친숙도**: 높음

### 특성
- 주문 접수와 서빙을 담당
- 빠른 주문 확인과 상태 변경이 필요
- 테이블 초기 설정을 수행할 수 있음
- 메뉴 관리 권한은 제한적

### 목표
- 새 주문을 빠르게 확인하고 싶다
- 주문 상태를 쉽게 변경하고 싶다
- 고객이 떠난 후 테이블을 빠르게 초기화하고 싶다

### 불편사항
- 새 주문을 놓치는 경우
- 테이블 초기화를 잊어버리는 경우
- 잘못된 주문을 처리하는 과정이 복잡한 경우

### 관련 스토리
- 매장 인증 (직원 계정)
- 실시간 주문 모니터링 (알림음)
- 주문 상태 변경
- 테이블 세션 종료
- 주문 삭제
